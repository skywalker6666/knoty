圖譜佈局策略：哪種適合 Knoty？

這是 Sprint 0 技術驗證的核心選擇，影響效能與使用體驗

A：靜態放射佈局（KnotyGraphV3 原型）

「我」固定在中心，直接關係人均勻排列在外圍。
位置確定、畫面穩定，SVG 渲染，不跑模擬。

優點

  • 原型已完成，直接整合真實資料
  • 手機效能佳（無物理模擬）
  • 使用者不會看到節點「跳來跳去」

缺點

  • 30+ 節點時外圈過擠
  • 無法展現自然聚落結構

B：D3 力導向模擬（force simulation）

節點互斥、邊吸引，自動形成聚落。相近圈子的人自然聚在一起。

優點

  • 聚落結構更直覺（社團、宿舍自然分群）
  • 30+ 節點仍有合理間距
  • CLAUDE.md spec 指定的方案

缺點

  • 啟動時節點會「晃動」到定位
  • 需驗證中低階機效能（Sprint 0 PoC 目標）
  • 實作較複雜，約需額外 2-3 天

Sprint 0 建議

如果目標是最快驗證資料連通 + 基本互動，選 A（靜態）先跑通，再 Sprint 1 換 B。
如果效能測試本身是 Sprint 0 的核心，選 B 直接驗證力導向效能。
